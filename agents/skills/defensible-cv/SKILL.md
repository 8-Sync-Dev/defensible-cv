# defensible-cv

## Role

This skill maintains a single source-of-truth CV (`cv_data.yaml`, rendercv v2.8 schema, English) in which every load-bearing number — downloads, citation counts, GitHub stars/forks, citing-paper page references — is mirrored in `data/research.json` and regenerated by `scripts/research.py`. The skill keeps that contract intact across edits, refreshes, and re-renders.

## Files owned

| Path | Purpose |
| --- | --- |
| `cv_data.yaml` | rendercv source. The only thing a recruiter ever sees (via PDF). |
| `scripts/research.py` | urllib + pypdf crawler. Refreshes downloads, citation counts, GitHub totals, and per-citing-paper PDF excerpts. |
| `data/research.json` | Machine-readable snapshot. Every number in `cv_data.yaml` MUST appear here. |
| `data/research.md` | Recruiter-readable summary built from `research.json`. |
| `data/.cache/` | 24h HTTP cache; `data/.cache/pdfs/` holds binary citing-paper PDFs. Gitignored. |
| `requirements.txt` | `rendercv[full]>=2.8`, `PyYAML>=6.0`, `pypdf>=4.0`. `scholarly` is optional and commented out. |
| `README.md` | Quickstart for the candidate. |
| `rendercv_output/` | Render artefacts (PDF, PNG, MD, HTML, Typst). Gitignored. |

## Mandatory rules

1. **No invented numbers.** Every figure in `cv_data.yaml` (downloads, citations, stars, forks, repo counts, citing-paper page numbers, reference labels) MUST be traceable to `data/research.json`. If a figure isn't there, run `scripts/research.py` first — never guess.
2. **No placeholder text in shipped copy.** Strings like `(live count in data/research.json)`, `sources tracked in …`, or `TODO` are forbidden in `cv_data.yaml`. Either inline the concrete number or remove the claim.
3. **Re-render after every content change.** A `cv_data.yaml` edit is not done until `uv run --with "rendercv[full]>=2.8" rendercv render cv_data.yaml` succeeds AND both rendered PNGs are visually checked (no broken `;`, no overflow to page 3, no truncated bold).
4. **2-page maximum.** Engineering resumes theme. If a change pushes to page 3, tighten copy or design knobs (`design.typography.line_spacing`, `design.sections.space_between_regular_entries`, `design.entries.highlights.space_between_items`) instead of dropping content.
5. **Markdown caveat: NEVER put `;` immediately after `**bold**`.** Typst's markdown lexer drops the punctuation. Use ` — `, `,`, or ` · ` instead. (This is a documented gotcha; see "Common rendering traps" below.)
6. **No `tel:` URI prefix on `cv.phone`.** rendercv v2.8 expects E.164-style `+CC-NNN-NNN-NNNN`. The display format is controlled by `design.header.connections.phone_number_format`.
7. **Folded scalars only for multi-line text fields.** Use YAML `>-` (folded) for summaries; never `|-` (literal) — the latter introduces hard line breaks that typst renders as separate paragraphs.
8. **Render via `uv`, not the project venv.** The canonical render command is `uv run --with "rendercv[full]>=2.8" rendercv render cv_data.yaml`. The `.venv/` is fine for the crawler but is not required for rendering.
9. **Crawler runs polite by default.** `scripts/research.py --no-scholar` is the default invocation in this skill. The `scholarly` (Google Scholar) dependency is optional and CAPTCHA-prone; only enable when the user explicitly asks for Scholar numbers.

## Standard workflow

1. **Refresh metrics first** (always, when a user request touches any number):
   ```bash
   uv run --with requests --with pyyaml --with pypdf \
     python scripts/research.py --no-scholar
   ```
   Outputs `data/research.{json,md}`. Use `--refresh` to bypass the 24h cache. Use `--print` to dump the recruiter summary to stdout.
2. **Inspect what changed.** Compare new vs old `data/research.json` figures (`git diff data/research.json`) before touching `cv_data.yaml`. If a citation count rose, decide whether it's load-bearing enough to surface in copy.
3. **Edit `cv_data.yaml`.** Update only the inline figures that moved. Keep the file's top comment about the regeneration contract. Respect Mandatory Rules 1–7.
4. **Render & visually verify.**
   ```bash
   rm -rf rendercv_output
   uv run --with "rendercv[full]>=2.8" rendercv render cv_data.yaml
   ```
   Confirm `rendercv_output/` has exactly two PNGs and one PDF. Open both PNGs; look for:
   - any `**bold**;` semicolon dropped,
   - publications block ending cleanly,
   - skills section landing on page 2 (no orphan page 3),
   - phone rendered in Vietnamese national format (`0703 930 513`).
5. **ATS sanity check.**
   ```bash
   pdftotext rendercv_output/Nguyễn_Phương_Anh_Tú_CV.pdf - | head -80
   ```
   Email, phone, dates, and section headings MUST be present and in reading order.

## Schema cheat-sheet (rendercv v2.8)

Entry types used in this CV:

- `experience_entry`: `company`, `position`, `start_date`, `end_date` (`YYYY-MM` or `present`), `location`, `summary?`, `highlights[]`
- `publication_entry`: `title`, `authors[]`, `doi`, `journal`, `date` (year), `summary?`
- `normal_entry` (used by Projects): `name` (may contain markdown link), `date`, `summary?`, `highlights[]`
- `education_entry`: `institution`, `area`, `degree`, `start_date`, `end_date`, `location`, `summary?`, `highlights[]`
- `one_line_entry` (used by Skills): `label`, `details`
- `summary`: list of strings

Header / contact:

- `phone`: `+CC-NNN-NNN-NNNN` (no `tel:` prefix). Display controlled by `design.header.connections.phone_number_format` (default: `national`).
- `email`: plain string; the rendered header links it as `mailto:`.
- `social_networks[].network`: enum — `GitHub`, `YouTube`, `ORCID`, `Google Scholar`, `LinkedIn`, etc. `username` is the raw handle (NO leading `@` for YouTube; rendercv prepends it on display).
- `website`: full `https://…` URL.

Design knobs proven to work with `engineeringresumes` for 2-page fit:

```yaml
design:
  theme: engineeringresumes
  typography:
    line_spacing: 0.55em        # default 0.6em
  sections:
    space_between_regular_entries: 0.32cm    # default 0.42cm
    space_between_text_based_entries: 0.10cm # default 0.15cm
  entries:
    summary:
      space_above: 0.05cm
    highlights:
      space_above: 0.05cm
      space_between_items: 0.05cm
```

## Common rendering traps

| Symptom | Cause | Fix |
| --- | --- | --- |
| `;` disappears after `**bold**` | typst markdown lexer drops orphan punctuation after closing `**` | Replace `;` with ` — `, `,`, or ` · ` |
| Multi-line paragraph splits into separate lines | YAML block scalar used `|-` instead of `>-` | Switch to `>-` (folded) |
| YouTube handle doubled (`@@Dev8Sync`) | Username field included a leading `@` | Drop the `@`; rendercv adds it |
| Google Scholar link is `…?user=` followed by garbage | Username field included `citations?user=` prefix | Either supply the raw Scholar author-ID or remove the entry |
| CV overflows to page 3 with mostly-empty page | One section grew | First, tighten copy. Second, lower the four spacing knobs above. Never drop a section. |
| Phone shows `tel:+84-703-930-513` | `tel:` URI prefix retained | Use `+84-703-930-513` (no scheme) |

## Research crawler — what each fetcher returns

(Reference, not for blind editing. The script is self-documenting; this table is for orientation.)

| Source | Function | Key fields in `data/research.json` |
| --- | --- | --- |
| OJS (jte.edu.vn) | `fetch_ojs_downloads` | `ojs.downloads_int`, `ojs.downloads_raw` |
| Semantic Scholar Graph | `fetch_semantic_scholar` | `semantic_scholar.citation_count`, `…influential_citation_count`, `…citing_papers[]` |
| OpenCitations COCI/Meta | `fetch_opencitations` | `opencitations.count`, `opencitations.citing_dois[]` |
| Crossref Event Data | `fetch_crossref_events` | `crossref_events.events[]` |
| Google Scholar (optional) | `fetch_google_scholar` | `google_scholar.citations` (or `available: false`) |
| GitHub Public API | `fetch_github` | `github.totals.{stars,forks,non_fork_repos}`, `github.top_repos[]` |
| Unpaywall → arXiv → pypdf | `enrich_citing_paper` | per citing paper: `pdf_status`, `reference_entry.{label,ref_pages}`, `contexts[].{page,snippet}` |

`pdf_status` legend:

- `ok` — PDF retrieved, our paper found in references AND in body. Inline excerpts available.
- `ref_only` — Found in references, no inline match (uncommon citation format).
- `no_match` — PDF retrieved but our paper not found in body. Likely false positive from S2.
- `not_oa` — Unpaywall reports no open-access version (typical for IEEE).
- `fetch_failed` — PDF URL exists but downloader was blocked (typical for MDPI behind Akamai, ACM dl.acm.org).

## When the user asks for…

| Request | Action |
| --- | --- |
| "Refresh the CV numbers" | Run crawler with `--no-scholar`, diff JSON, update any inline numbers that moved in `cv_data.yaml`, re-render. |
| "Add a new job / publication / project" | Append the entry. If it references any external metric (stars, downloads, citations), first verify the metric appears in `data/research.json`; if not, extend `scripts/research.py` (add a fetcher) and refresh. |
| "Translate to Vietnamese" | Out of scope for this skill — current decision is English-only. |
| "Switch theme" | Render once with the new theme, then re-tune the four spacing knobs to keep two pages. |
| "It overflows to 3 pages" | Apply the design-knob ladder before cutting content. |

## Communication

Code and `cv_data.yaml` content stay English. User-facing replies follow the user's language (Vietnamese is typical here).
